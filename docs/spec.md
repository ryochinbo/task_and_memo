# 看板方式タスク管理ツール 仕様書

## 1. システム概要

### 1.1 目的
ドラッグ&ドロップで操作できるシンプルな看板方式のタスク管理ツールを提供し、チームまたは個人のタスク管理を効率化する。

### 1.2 システム構成
- **フロントエンド**: HTML + CSS + JavaScript
- **バックエンド**: Python Flask
- **データ保存**: JSONファイル + Gitバージョン管理
- **稼働環境**: ローカルサーバー（http://127.0.0.1:5000）

## 2. 機能仕様

### 2.1 タスク管理機能

| 機能 | 説明 |
|------|------|
| タスク作成 | 新規タスクの追加 |
| タスク編集 | 既存タスクの情報更新 |
| タスク削除 | タスクの削除（確認ダイアログ付き） |
| 状態変更 | ドラッグ&ドロップで状態を移動 |

### 2.2 タスクデータ項目

| 項目名 | データ型 | 必須 | 説明 |
|--------|----------|------|------|
| id | string | ◯ | UUID（一意識別子） |
| name | string | ◯ | タスク名 |
| status | string | - | 状態（todo/in_progress/done） |
| memo | string | - | メモ（自由記述） |
| priority | string | - | 重要度（high/medium/low） |
| tags | array | - | タグリスト |
| tag_custom | string | - | タグ自由記述 |
| has_ball | boolean | - | ボールが自分にあるか |
| ball_holder | string | - | ボール保持者名 |
| due_date | string | - | 締め切り（ISO 8601形式） |
| created_at | string | ◯ | 作成日時（ISO 8601形式） |
| start_date | string | - | 開始日（ISO 8601形式） |

### 2.3 状態定義

| 状態 | 説明 | カラム表示 |
|------|------|-----------|
| todo | 未着手のタスク | 📝 TODO |
| in_progress | 作業中のタスク | 🚧 進行中 |
| done | 完了したタスク | ✅ 完了 |

### 2.4 重要度定義

| 重要度 | 表示 | 優先度 |
|--------|------|--------|
| high | 🔴 高 | 高 |
| medium | 🟡 中 | 中 |
| low | 🟢 低 | 低 |

## 3. API仕様

### 3.1 エンドポイント一覧

| メソッド | パス | 説明 |
|----------|------|------|
| GET | / | トップページ |
| GET | /api/tasks | 全タスク取得 |
| POST | /api/tasks | タスク作成 |
| PUT | /api/tasks/{id} | タスク更新 |
| DELETE | /api/tasks/{id} | タスク削除 |

### 3.2 API詳細

#### GET /api/tasks
全タスクを取得する。

**リクエスト**: なし

**レスポンス**:
```json
{
  "tasks": [
    {
      "id": "uuid",
      "name": "タスク名",
      "status": "todo",
      "memo": "メモ",
      "priority": "medium",
      "tags": ["タグ1", "タグ2"],
      "tag_custom": "",
      "has_ball": false,
      "ball_holder": "担当者名",
      "due_date": "2025-12-31",
      "created_at": "2025-01-01T00:00:00",
      "start_date": "2025-01-01"
    }
  ]
}
```

#### POST /api/tasks
新規タスクを作成する。

**リクエストボディ**:
```json
{
  "name": "タスク名",
  "status": "todo",
  "priority": "medium",
  "memo": "メモ",
  "tags": ["タグ1"],
  "tag_custom": "",
  "has_ball": false,
  "ball_holder": "",
  "start_date": "",
  "due_date": ""
}
```

**レスポンス**: 作成されたタスク（201 Created）

#### PUT /api/tasks/{id}
タスクを更新する。

**リクエストボディ**: POST /api/tasks と同じ

**レスポンス**: 更新されたタスク

#### DELETE /api/tasks/{id}
タスクを削除する。

**レスポンス**: 削除されたタスク

## 4. データ永続化

### 4.1 データファイル
- **場所**: `data/tasks.json`
- **形式**: JSON
- **エンコーディング**: UTF-8

### 4.2 Git自動保存
以下のタイミングで自動的にgit commit & pushが実行される：
- タスク作成時
- タスク更新時
- タスク削除時

**コミットメッセージ**: `Update tasks - {日時}`

### 4.3 Git操作対象
- `data/` ディレクトリが独立したGitリポジトリとして管理される

## 5. UI仕様

### 5.1 レイアウト
- 3カラムの看板ボード（TODO、進行中、完了）
- ヘッダーにタイトルと新規タスクボタン
- レスポンシブデザイン対応

### 5.2 カードデザイン
- タスク名、メモ、タグ、メタ情報を表示
- 重要度に応じた色分け（左ボーダー）
- 編集・削除ボタン

### 5.3 インタラクション
- ドラッグ&ドロップで状態変更
- モーダルダイアログで編集
